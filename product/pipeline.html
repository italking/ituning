Tuning Pipeline

Tuning plan

For clusters that need to be tuned for JVM parameters, TMaestro generates a tuning plan for the cluster, each of which contains a tuning pipeline, and a cluster can only have one tuning plan in execution.

Step 1 Health Testing

TMaestro performs health checks on the cluster to detect potential problems with the cluster, checking response time, YGC count, YGC time, FGC count, FGC time, heap memory, Old area usage, garbage collector, and more.

Step 2 Expected assessment

TMaestro evaluates the cluster’s expectations, performs a comprehensive analysis through an algorithm, and selects a sample machine for tuning tests to assess how the cluster will affect if the parameters are adjusted

Step 3 Tuning process

TMaestro analyzes all the JVM parameters that need to be adjusted, and if the user needs to modify a specific parameter, it can be manually corrected here. In the call process, TMaster will optimize the cluster single machine in batches, after each batch tuning is completed, intelligent risk detection will be carried out, if a problem is detected, the entire cluster tuning will be rolled back to ensure tuning security.

Step 4 Effect verification

TMaestro will cluster monitoring indicators for pre- and post-tuning comparative analysis, evaluate the actual tuning effect, if the effect is not obvious, it will be fed back to the algorithm center for sample learning evolution
